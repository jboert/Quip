# Gitignore Generated Info.plist Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Stop tracking `QuipMac/Info.plist` and `QuipiOS/Info.plist` in git so they can't be edited by hand and silently clobbered by the next `xcodegen generate`. Pre-flight drift check ensures no hand-edits are lost during the untracking.

**Architecture:** Configuration-only change (git index + `.gitignore` + `README.md`). No runtime code is touched. Work is bounded to a single commit on `eb-branch` after a drift-detection safety gate.

**Tech Stack:** `git`, `xcodegen`, `.gitignore`, Markdown (`README.md`). No compiled code or tests.

**Spec:** `docs/superpowers/specs/2026-04-15-gitignore-generated-info-plist-design.md`

**Worktree note:** This plan is small enough (one commit, zero code) that a dedicated worktree is not necessary. Execute directly on `eb-branch`.

## Task 1: Pre-work drift verification (safety gate)

**Purpose:** Confirm that the currently-tracked `Info.plist` files are byte-identical to what `xcodegen generate` would produce right now. If drift exists, it represents a hand-edit that would be destroyed by the untracking — that edit must be moved to `project.yml` in a separate preceding commit, and the rest of this plan pauses.

**Files:**
- Read-only inspection: `QuipMac/Info.plist`, `QuipiOS/Info.plist`
- Runs: `xcodegen` in both `QuipMac/` and `QuipiOS/`

- [ ] **Step 1.1: Verify working tree is clean**

Run:
```bash
git status
```

Expected: `nothing to commit, working tree clean`. If not clean, stop and clean up first — you need a known-good baseline to detect drift.

- [ ] **Step 1.2: Run xcodegen in QuipMac**

Run:
```bash
cd QuipMac && xcodegen generate && cd ..
```

Expected: xcodegen prints its progress lines and exits 0. If xcodegen is not installed, stop and install via `brew install xcodegen`.

- [ ] **Step 1.3: Run xcodegen in QuipiOS**

Run:
```bash
cd QuipiOS && xcodegen generate && cd ..
```

Expected: xcodegen prints its progress lines and exits 0.

- [ ] **Step 1.4: Diff Info.plist files for drift**

Run:
```bash
git diff -- QuipMac/Info.plist QuipiOS/Info.plist
```

Expected: **empty output** (zero diff). This means `xcodegen` regenerated byte-identical files to the currently-tracked versions, confirming no hand-edits will be lost by untracking.

**If the diff is non-empty:** STOP. A hand-edit exists in one of the tracked files that is not reflected in `project.yml`. Do not proceed with the rest of this plan. Instead:

1. Inspect the diff to understand what the hand-edit was trying to achieve.
2. Move the hand-edit's intent into the appropriate `project.yml` `info.properties` section as a separate commit.
3. Re-run this task from Step 1.2.
4. Only resume Task 2 after Step 1.4 produces an empty diff.

- [ ] **Step 1.5: Discard xcodegen's incidental writes to `.xcodeproj`**

`xcodegen` may have regenerated `*.xcodeproj/project.pbxproj` with identical content but different timestamps, or with unrelated cosmetic drift. Confirm the state:

Run:
```bash
git status
```

Expected: Either clean, or a non-empty list of `.xcodeproj/project.pbxproj` modifications that are unrelated to this plan. If the latter, restore them (they're not part of this change):

Run:
```bash
git checkout -- QuipMac/QuipMac.xcodeproj QuipiOS/QuipiOS.xcodeproj
```

Then re-confirm:
```bash
git status
```

Expected: `nothing to commit, working tree clean`.

## Task 2: Untrack the Info.plist files

**Purpose:** Remove both Info.plist files from the git index without deleting them from the working tree.

**Files:**
- Stage removals: `QuipMac/Info.plist`, `QuipiOS/Info.plist`

- [ ] **Step 2.1: Remove both files from the index**

Run:
```bash
git rm --cached QuipMac/Info.plist QuipiOS/Info.plist
```

Expected output:
```
rm 'QuipMac/Info.plist'
rm 'QuipiOS/Info.plist'
```

- [ ] **Step 2.2: Verify files are still on disk**

Run:
```bash
ls -l QuipMac/Info.plist QuipiOS/Info.plist
```

Expected: both files exist with non-zero size. `git rm --cached` only removes from the index, not from the working tree.

- [ ] **Step 2.3: Verify the index state**

Run:
```bash
git status
```

Expected output contains (among possibly other things):
```
Changes to be committed:
  (use "git restore --staged <file>..." to unstage)
	deleted:    QuipMac/Info.plist
	deleted:    QuipiOS/Info.plist

Untracked files:
  (use "git add <file>..." to include in what will be committed)
	QuipMac/Info.plist
	QuipiOS/Info.plist
```

This is the intended state: the index now considers them deleted, and the working-tree copies are "new untracked files" from git's perspective. The `.gitignore` update in Task 3 will make the untracked-files section drop them.

## Task 3: Add .gitignore entries

**Files:**
- Modify: `.gitignore`

- [ ] **Step 3.1: Edit .gitignore (explicit find-and-replace)**

Use the Edit tool to make this replacement in `.gitignore`:

**Find** (exactly this block, including the blank line between `build/` and `# Swift Package Manager`):

```
# Xcode
xcuserdata/
*.xcuserstate
DerivedData/
build/

# Swift Package Manager
```

**Replace with**:

```
# Xcode
xcuserdata/
*.xcuserstate
DerivedData/
build/

# xcodegen-generated Info.plist — regenerated from <project>/project.yml, see README
QuipMac/Info.plist
QuipiOS/Info.plist

# Swift Package Manager
```

This keeps the existing `# Xcode` block intact, inserts the new `# xcodegen-generated Info.plist` section right after it, and preserves the blank-line-before-next-section convention.

- [ ] **Step 3.2: Verify .gitignore now contains the new entries**

Run:
```bash
grep -n "Info.plist" .gitignore
```

Expected: two lines of output, one for `QuipMac/Info.plist` and one for `QuipiOS/Info.plist`. Line numbers will reflect the insertion point.

- [ ] **Step 3.3: Verify the untracked-files section drops the Info.plists**

Run:
```bash
git status
```

Expected: the "Untracked files" section should no longer list `QuipMac/Info.plist` and `QuipiOS/Info.plist`. The "Changes to be committed" section should still show them as `deleted:`, and there should be a new `modified: .gitignore` entry.

## Task 5: Pre-commit verification

**Purpose:** Confirm everything is in the expected state before committing, including that a fresh `xcodegen` run does not re-introduce tracked Info.plist changes.

- [ ] **Step 5.1: Run xcodegen again to confirm gitignore works**

Run:
```bash
cd QuipMac && xcodegen generate && cd ..
cd QuipiOS && xcodegen generate && cd ..
```

Expected: both succeed. `Info.plist` files on disk get rewritten by xcodegen, but because they are now `.gitignore`'d, git should not see them as changes.

- [ ] **Step 5.2: Verify gitignore is taking effect**

Run:
```bash
git status
```

Expected state of `git status`:
```
Changes to be committed:
	deleted:    QuipMac/Info.plist
	deleted:    QuipiOS/Info.plist
	modified:   .gitignore
	modified:   README.md
```

There should be **no** entries for `QuipMac/Info.plist` or `QuipiOS/Info.plist` in any "Changes not staged for commit" or "Untracked files" section — if there are, `.gitignore` is not catching them and Task 3 needs to be revisited.

- [ ] **Step 5.3: Verify .xcodeproj files did not get incidentally modified**

Run:
```bash
git status --porcelain | grep -v -E "^(D  QuipMac/Info\.plist|D  QuipiOS/Info\.plist|.M \.gitignore|.M README\.md)$"
```

Expected: empty output (no lines). If there are other entries, the most likely cause is xcodegen touching a `*.xcodeproj/project.pbxproj` — discard those with `git checkout -- QuipMac/QuipMac.xcodeproj QuipiOS/QuipiOS.xcodeproj` and re-run this step.

- [ ] **Step 5.4: Stage the four changes**

Run:
```bash
git add .gitignore README.md QuipMac/Info.plist QuipiOS/Info.plist
```

Note: `git add` on a file being removed from the index will finalize the removal, which is what we want here.

- [ ] **Step 5.5: Review the staged diff one last time**

Run:
```bash
git diff --cached --stat
```

Expected output (file names and line counts — numbers may shift by 1–2 if the files change before execution):
```
 .gitignore         |  4 ++++
 QuipMac/Info.plist | 36 ------------------------------------
 QuipiOS/Info.plist | 62 --------------------------------------------------------------
 README.md          |  2 ++
 4 files changed, 6 insertions(+), 98 deletions(-)
```

If the deletion counts differ substantially (more than a handful of lines), inspect `git diff --cached` to understand why — the most likely cause is an `xcodegen` run between plan-writing and execution that changed the generated contents, which is benign but worth eyeballing.

ls -l QuipMac/Info.plist QuipiOS/Info.plist

Open both `QuipMac/QuipMac.xcodeproj` and `QuipiOS/QuipiOS.xcodeproj` in Xcode and build each scheme (Cmd+B). Alternatively, from the command line:

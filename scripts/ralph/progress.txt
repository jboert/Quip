## Codebase Patterns
- macOS project builds with: `xcodebuild -project QuipMac/QuipMac.xcodeproj -scheme QuipMac -configuration Debug build`
- QuipLinux cannot be typechecked on macOS — requires GTK/GLib system libs only available on Linux. Verify Rust changes are type-safe by inspection.
- AppleScript string escaping uses `escapeForAppleScript()` in KeystrokeInjector.swift — only escapes `\` and `"` (sufficient for `keystroke`, NOT for `do script`)
- Terminal.app text injection uses System Events keystrokes; iTerm2 uses `write text` (iTerm2's API is safe as it doesn't interpret shell)
- Multi-line text for Terminal.app must be split on newlines, each line typed with `keystroke`, then `key code 36` (Return) between lines
- Rust protocol messages: use `Deserialize` only for incoming (client→server), `Serialize` only for outgoing (server→client). Use `#[serde(skip_serializing_if = "Option::is_none")]` for optional fields.
- Protocol changes touch 5 files: Shared/MessageProtocol.swift, Protocol.kt, messages.rs, protocol.md, MessageProtocolTests.swift
- Android Gradle build is broken (JDK version mismatch) — pre-existing, not caused by protocol changes
- New services: add file to Xcode project.pbxproj (PBXBuildFile, PBXFileReference, PBXGroup children, PBXSourcesBuildPhase) and inject via `.environment()` in QuipMacApp.swift
- Keychain storage pattern: use `kSecClassGenericPassword` with `kSecAttrAccount` as key, `kSecAttrAccessibleWhenUnlockedThisDeviceOnly` for sensitive data
- PINManager is `@Observable` — auto-generates on first launch, stores in Keychain, accessed via `@Environment(PINManager.self)`
- WebSocketServer auth: connections start unauthenticated, only "auth" messages processed until authenticated. Set `pinManager` property before calling `start()`.
- Per-connection send uses private `send(_:to:)` method; broadcast only goes to authenticated clients.
- Linux WsServer uses HashMap<usize, ClientConnection> with per-connection auth state (not Vec<WsSink>)
- PINManager is shared via Clone (Arc<RwLock<String>> interior) — same instance for UI and WsServer
- Settings dialog `show_settings` requires `&PINManager` parameter
- Android network_security_config.xml cannot express IP CIDR ranges — use code-level validation (NetworkValidator.kt) for private IP restrictions
- PIN stored at XDG_CONFIG_HOME/quip/pin (defaults to ~/.config/quip/pin), file 0600, dir 0700
- iOS auth flow: connection established → onAuthRequired callback → PIN entry UI → sendAuth(pin:) → auth_result handler. Cached PIN auto-sent on reconnect.
- iOS UI state gates: use `client.isAuthenticated` (not `isConnected`) for enabling controls, landscape mode, and keyboard toggle
- Android auth flow: connect → onAuthRequired callback → PIN entry UI (MainScreen) → sendAuth() → auth_result. Cached PIN auto-sent on reconnect. Controls gated on `isAuthenticated`.
- Android UI state: `showPinEntry` controls PIN screen visibility; `isAuthenticated` gates LayoutScreen and landscape orientation
- macOS tunnel log path: `~/Library/Caches/Quip/tunnel.log` (via FileManager cachesDirectory). Dir 0700, file 0600.
- SecretRedactor (macOS) is a static enum with regex patterns; Linux equivalent is `secret_redactor::redact()` using `LazyLock<Vec<RedactPattern>>`
- Linux tunnel log path: `XDG_CACHE_HOME/quip/tunnel.log` (defaults to `~/.cache/quip/tunnel.log`). Dir 0700, file 0600. Uses `prepare_tunnel_log()` helper.

- **Learnings for future iterations:**
  - Android's network_security_config.xml `<domain>` cannot express IP CIDR ranges — `includeSubdomains` only works for DNS subdomain hierarchy, not IP subnets
  - Setting `base-config cleartextTrafficPermitted="false"` blocks OkHttp cleartext connections to ALL non-whitelisted hosts (checked via `NetworkSecurityPolicy.isCleartextTrafficPermitted(hostname)`) — there is no way to whitelist IP ranges in XML
  - For apps that connect to local IPs via ws://, the security enforcement must be split: XML for domain-based restrictions, code for IP-range restrictions
  - `InetAddress.isSiteLocalAddress` covers exactly RFC 1918 ranges (10/8, 172.16/12, 192.168/16); `isLoopbackAddress` covers 127/8; `isLinkLocalAddress` covers 169.254/16
  - OkHttp 4.12.0 normalizes ws:// to http:// internally, so the platform cleartext check applies to WebSocket connections

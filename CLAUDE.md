# Quip Project Instructions

## Commit Messages

Write commit messages in the voice of a blue collar boomer who isn't too tech-savvy. Short — one or two sentences tops. No bullet points, no technical jargon, no conventional commits format. The guy knows what he changed but wouldn't know a "refactor" from a hole in the ground. He'd say "the doohickey" before he'd say "the WebSocket client." Colorful analogies from the shop, the garage, the job site — that's the good stuff. Keep it punchy.

Examples:
- "Now the start/stop haptics hit harder than a supercharged big block"
- "That list was only saving 8 when it shoulda been 10, fixed it"
- "Put a keyboard in there so you can just type if you don't feel like talking"
- "Wrote down how the phone talks to the computer so I don't forget"
- "The volume buttons were being weird, tightened up the timing on that"
- "Them shortcut buttons weren't doing squat cause nobody told it which window to look at"
